- **分层架构**：遵循 Handler-Service-Repository 经典三层结构。`handler.go` 负责 HTTP 请求解析与响应映射；`service.go` 处理核心业务逻辑（密码比对、JWT 签发/验证）；`repository.go` 基于 `pgxpool` 执行原生 SQL 查询。
- **中间件集成**：通过 `middleware.go` 实现 `AuthMiddleware`，拦截受保护路由，解析 Bearer Token 并将用户上下文注入 `gin.Context`。
- **数据模型**：`model.go` 定义了请求/响应 DTO 及数据库实体（User, Role），支持用户-角色-权限的多对多关联查询。
- **依赖方向**：Handler 依赖 Service，Service 依赖 Repository，Repository 依赖底层 PostgreSQL 连接池，依赖方向清晰单向。
