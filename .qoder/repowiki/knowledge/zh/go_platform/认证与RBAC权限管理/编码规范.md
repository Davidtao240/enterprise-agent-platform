- 统一错误映射：Service 层定义预置错误（如 ErrInvalidCredentials），Handler 层通过 errors.Is 判断并映射为特定 HTTP 状态码（401/403）。
- 安全响应规范：登录失败时统一返回模糊错误信息，避免泄露用户名是否存在等敏感细节。
- 上下文注入模式：鉴权中间件验证通过后，将 user_id 和 username 存入 gin.Context，后续 Handler 通过 GetString 获取而非重复解析 Token。
- 原生 SQL 实践：Repository 层直接使用 pgxpool.QueryRow/Query 执行参数化 SQL，保持查询透明性并避免 ORM 开销。
