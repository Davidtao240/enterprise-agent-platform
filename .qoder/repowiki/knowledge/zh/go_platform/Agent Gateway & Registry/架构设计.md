- **分层架构**：采用经典的 Controller-Service-Repository 模式。`Handler` 负责 HTTP 路由与参数校验，`Gateway` 封装核心业务逻辑（图验证、域策略检查、远程调用），`Repository` 处理 PostgreSQL 数据持久化。
- **核心网关流程**：`Gateway.Execute` 实现了严格的调用链：验证 `graph_key` 活性 -> 校验 `domain_policy` 域隔离 -> 记录初始运行日志 -> 发起对 Python Agent Service 的 HTTP POST 请求 -> 更新最终状态与用量审计。
- **依赖注入与解耦**：通过 `gatewayRepository` 和 `handlerRepository` 接口定义依赖边界，支持在测试中使用 `fakeGatewayRepo` 等模拟对象，实现了业务逻辑与数据存储的解耦。
- **审计集成**：深度集成 `audit.Repository`，在网关执行的关键节点（开始、完成、失败）及审批操作（通过、驳回）中自动写入审计日志，确保操作可追溯。
